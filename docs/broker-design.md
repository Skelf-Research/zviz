# Broker Design

The Zig broker is a tiny policy kernel that mediates a small set of syscalls using seccomp user notification. It replaces broad syscall interposition with targeted, verifiable decisions.

## Responsibilities

- Receive `SECCOMP_RET_USER_NOTIF` events.
- Evaluate policy for the syscall and its arguments.
- Return a result or deny deterministically.
- Emit audit-friendly decision logs.

## Brokered syscall set (initial)

- `openat` and `openat2`
- Selected `ioctl` ranges
- Socket creation variants that require fine-grained policy

Exact sets are profile-specific and generated by the policy compiler.

## Broker contract

- Never do a “check path then open path” flow.
- Prefer `openat2` with `RESOLVE_*` flags to prevent path traversal and symlink tricks.
- Return file descriptors via `SCM_RIGHTS`, not paths.
- Use `O_PATH` + `fstat` for preflight checks when necessary.

## TOCTOU resistance

- Resolve and open in one kernel operation (`openat2`).
- Avoid user-controlled path re-evaluation.
- Reject ambiguous or unsupported path resolutions.

## ioctl strategy

- Default deny for unknown ioctl commands.
- Allow only specific ioctl numbers and argument shapes per profile.
- Maintain per-subsystem allowlists (for example, TTY vs GPU vs networking).

## Network mediation

- Prefer namespace-level enforcement and eBPF/iptables rules.
- Use the broker only for uncommon socket operations that need explicit policy decisions.

## Auditing

Each decision should include:

- syscall number and arguments
- policy rule ID
- allow/deny outcome
- latency and error code
